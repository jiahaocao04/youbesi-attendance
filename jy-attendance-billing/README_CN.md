# 优贝思校外托管中心考勤收费系统（最小可靠版）

这个系统的目标是：以每天手机点名考勤为基础，月底自动生成应缴金额、欠费名单、充值盈余和每个学生缴费明细。

核心原则：

- Excel 只作为导出结果，不作为多人同时编辑的数据库。
- 老师只记录事实：A=午餐出勤，P=下午/晚托出勤；A+P=全天；都不勾=未出勤不收费。
- 特殊情况直接写入“事件提醒”，有内容的学生行会淡红显示，方便当天注意。
- 学生个人购物记录在点名表里录入“物品 + 金额”，月底自动进入该学生应缴金额。
- 金额由系统根据考勤和收费规则自动计算。
- 余额不允许直接手改，必须通过充值、退款、调整或月结产生。
- 每次重要操作都会留下操作人和时间，方便追溯。

## 如何启动

双击或在 PowerShell 里运行：

```powershell
.\start.ps1
```

启动后电脑浏览器打开：

```text
http://127.0.0.1:8766
```

如果老师手机和这台电脑在同一个 Wi-Fi 下，可以用启动窗口里显示的局域网地址访问，例如：

```text
http://192.168.1.23:8766
```

第一次打开后，先点“导入现有学生名单”。系统会读取之前整理好的 94 名学生和期初余额。

## 登录和权限

初始管理员账号：

```text
用户名：admin
PIN：123456
```

第一次进入后，请到“账户管理”里把管理员 PIN 改掉，并给每位老师创建独立账号。

权限规则：

- 管理员：可查看学生名单、充值登记、月底结算、导出报表、成本记录、账户管理，也可以点名。
- 老师：只能进入“手机点名”，不能查看充值、月结、导出、账户管理。
- 后端接口也会校验权限，不只是前端隐藏按钮。

## 多设备使用

电脑作为主机运行系统；老师手机和电脑连同一个 Wi-Fi 后，在手机浏览器打开首页显示的局域网地址，例如：

```text
http://192.168.1.23:8766
```

为了真实使用稳定，老师端做了这些保护：

- 保存时会提交当天整张点名表，让未勾 A/P 的学生明确成为“未出勤”。
- 点名未保存时会显示“未保存”状态。
- 断网或误关页面前，本机会保存草稿。
- 保存中按钮会锁定，避免重复提交。
- 老师界面只渲染当前变化的行，减少手机端跳动和卡顿。

## 第一版功能

1. 学生名单导入
2. 自动载入统一手机点名册
3. 表格式 A/P 点名
4. 当日未到提醒子页面和事件提醒
5. 学生个人购物登记并进入月结
6. 充值登记
7. 月底自动结算
8. 导出欠费名单和学生缴费明细 Excel/PDF
9. 管理员成本记录和成本 PDF 报表

## 数据位置

系统数据库保存在：

```text
jy-attendance-billing\data\jy.db
```

这个文件非常重要，后续真实使用时需要定期备份。

PDF 报表会自动保存到：

```text
优贝思报表归档
```

下面会按类型分文件夹：

```text

月报
学生出勤报表
成本报表
```

## 当前收费规则

- 单独午餐：30 元 / 天
- 全天托管：50 元 / 天，包含午餐 + 晚托/托管
- 单独晚托：25 元 / 天
- 床位费：50 元 / 月
- 按学期提前充值的学生：免床位费

这些可以在“月底结算”页面修改。

## 出勤和计费规则

只有 A/P 被勾选的服务才计费；未勾选不计费。

系统按每天点名组合自动判断收费项目：

- 只勾 A：按单独午餐 30 元 / 天
- A + P 都勾：按全天托管 50 元 / 天
- 只勾 P：按单独晚托 25 元 / 天
- A/P 都不勾：当天不收费
- 学生购物：按当天录入的购物金额额外计入该学生本月应缴
- 预充值大于实际扣费：剩余金额自动作为下月余额

管理员进入“手机点名”时，默认看到点名册；“未到提醒”会作为单独小子页面显示人数，点开后查看未到名单。

## 报表规则

- 每次保存考勤后，系统自动生成当天“每日考勤报表 PDF”并保存到本地。
- 管理员生成/重算月结后，系统自动生成当月“月度结算报表 PDF”并保存到本地。
- 管理员可以在“导出报表”里一键生成某个学生的单独出勤 PDF，详细到每天。
- 学生单独出勤 PDF 会列出每天的事件提醒和购物明细。
- 管理员可以在“成本记录”里登记房租、水电、菜、肉、水果、牛奶、人工、其他成本，并生成月度成本 PDF。
- 报表用于追溯和核对；实际收费仍以系统数据库和月结计算为准。
